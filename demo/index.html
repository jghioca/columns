Columns Demo

Usage

Example 1: Let's say you want to create the 2 columns shown below, which have equal widths and the class names of col1 and col2.

.col1

.col2

In your SCSS file you can do this by simply writing: 

.col1, .col2 {
  @include column(1, 2);
}

The first argument of column() that we pass in is the numerator and the second argument we pass in is the denominator. In other words, one column takes up 1 out of 2 spots (1/2).

The actual output of '@include column(1, 2)' after compilation is: 

float: left;
clear: right;
width: 50%;

The columns are positioned using 'float: left'. The 'clear: right' declaration indicates the right side of an element's box may not be adjacent to an earlier floating box. And the width value is calculated based on the numbers you passed in. Note that this mixin outputs all CSS units as a percentage to keep the columns fluid for any screen size.

Example 2: Let's say you want to create the 3 columns shown below with a 5% margin in between the columns.

.col1

.col2

.col3

In your SCSS you can do this by simply writing: 

.col1, .col2, .col3 {
  @include column(1, 3, 5%);
}

.col3 {
  @include last();
}

The first argument of column() that we pass in is 1, the numerator, and the second argument we pass in is 3, the denominator. In other words, one column takes up 1 out of 3 spots (1/3). The third argument we pass in is the margin that we want to have between the columns.

Notice that it has to be a percentage because the mixin uses helpers to make calculations and any math operations involving different units can't be done. Also, we don't want to have a margin-right after the last column because it would not be flush. So we can add '@include last()' to the last column in order for it to get a margin-right of 0 or you can just write 'margin-right: 0'. It's up to you.

Example 3: Let's say you want to create 2 columns where one is bigger than the other and you need one to be a specific percentage, but you don't know how to represent it with a simple fraction. That specific percentage needs to be 64.4% for arguments sake.

.col1

.col2

You can actually write your SCSS like this: 

.col1 {
  @include column(64.4, 100);
}

.col2 {
  @include column(35.6, 100);
}

You can just plug in the percentage you need as the first argument and have it be out of 100. I calculated the remaining space for the second column I have available with subtraction (100% - 64.4% = 35.6%).

.col5

Here is the magical SCSS: 

.col1 {
  @include column(5, 15);
}
.col2 {
  @include column(2, 15);
}
.col3 {
  @include column(3, 15);
}
.col4 {
  @include column(4, 15);
}
.col5 {
  @include column(1, 15);
}

If you take all the numerators and add them up, you will notice that the sum equals the denominator (15 in this example).

You can use this mixin with any type of consistently spaced column layout you need. There are unlimited layout combinations you can have for your needs.
